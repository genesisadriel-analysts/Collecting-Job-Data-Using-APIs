Project: Collecting Job Data Using APIs
This project demonstrates how to programmatically interact with REST APIs to retrieve, process, and analyze professional job market data. It transitions from basic API connectivity tests using space exploration data to a focused analysis of technical roles and geographic demand.

### Core Features
API Data Retrieval: Uses the requests library to fetch live JSON data from external web services.

Technical Skill Analysis: Features automated functions to count job postings for specific technologies like Python, Java, and SQL.

Geographic Filtering: Identifies job availability across major US hubs, including Washington DC and Los Angeles.

Data Export: Utilizes pandas and openpyxl to structure raw JSON data into organized Excel spreadsheets (.xlsx).

### Dataset Description
The dataset is sourced from an IBM Cloud endpoint and includes comprehensive professional metadata:

Job Title: Official designation of the role.

Key Skills: Technical competencies required (e.g., Python, C++, MySQL).

Location: The specific city for the posting.

Metadata: Includes details on Industry, Role Category, and Experience Required.

### Technical Stack
Language: Python 3.14+

Libraries: requests, pandas, openpyxl, PIL, json

Environment: Jupyter Notebook
